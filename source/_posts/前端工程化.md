# 前端工程化
了解前端工程化从认识「编译原理」开始，编译本质就是把「一些字符串」转换成「另外一些字符串」
编译原理步骤：
1. 词法分析(sourceCode -> tokens) 
2. 语法分析(tokens -> ast)
3. 代码转换(ast -> newAst)
4. 代码生成(newAst -> newCode)
基于以上我们举一些常见的case：
1. Typescript：Typescript代码 -> Javascript代码
2. babel：ES6语法 -> 有兼容性的语法（polyfill）
3. rollup：多文件代码 -> 单文件代码（依赖分析+TreeShaking）
4. webpack：一个万物皆打包的架构，支持通过各种loader打包各种各样的资源（css-loader、vue-loader、图片字体等），同时提供热更新、本地开发、懒加载、分包等能力，真正进入工程化时代。
5. esbuild：也是打包架构。用go写的，比webpack更快，但生态不如webpack。
6. vite：相比webpack，主打极快启动和更好的热更新机制。
7. vue、react、跨端：本质也是编译，tempalte -> vue代码，jsx -> React代码，跨端 -> 对应平台可运行的代码
